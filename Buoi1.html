Java là gì?

  • Ngôn ngữ lập trình bậc cao, hướng đối tượng (OOP), do Sun Microsystems (nay là Oracle) phát triển.
  • Chạy trên JVM (Java Virtual Machine) với bytecode → đa nền tảng.
  • Hệ sinh thái phong phú: thư viện chuẩn, build tools, framework.
Mấu chốt: biên dịch sang bytecode, chạy trên JVM nên ít phụ thuộc HĐH.

Đặc điểm nổi bật

  • Bảo mật, ổn định: kiểm tra kiểu mạnh
  • Hiệu năng tốt: JIT compilation

JVM, JRE, JDK khác nhau thế nào?

JVM → JRE = JVM + Runtime → JDK = JRE + Tools
🖥️

JVM

Máy ảo thực thi bytecode, có GC/JIT.

📦

JRE

JVM + thư viện runtime để chạy ứng dụng.

🛠️

JDK

JRE + công cụ phát triển (javac, javadoc...).

.java → javac → .class (bytecode) → JVM

Lập trình viên cần JDK để biên dịch và chạy dự án.

Cài đặt JDK (Windows)

  1. Tải JDK LTS: 17 hoặc 21 (Oracle/OpenJDK/Temurin).
  2. Cài đặt và chọn “Set JAVA_HOME” nếu có.
  3. Thiết lập biến môi trường khi cần: JAVA_HOME, thêm %JAVA_HOME%\bin vào PATH.
  4. Kiểm tra phiên bản:
java -version
javac -version

Nếu xuất hiện phiên bản, môi trường đã sẵn sàng.

Tạo Project đầu tiên

  • Tạo file Main.java
  • Cấu trúc thư mục tối thiểu:
project/
└── src/
    └── Main.java

Tên file nên trùng với tên lớp public chứa main.

Hello World

    public static void main(String[] args) {
        System.out.println("Xin chào, Java!");
    }

Chạy trong IDE (Run) hoặc dùng dòng lệnh.

Biên dịch & chạy bằng dòng lệnh

// Từ thư mục chứa src/Main.java
cd src
javac Main.java     // biên dịch → sinh Main.class
java Main           // chạy chương trình

Không cần đuôi .class khi chạy bằng java.

Cấu trúc cơ bản

  • Lớp (class): đơn vị tổ chức chính
  • Hàm main: điểm vào chương trình
  • Khối lệnh: dùng dấu ngoặc nhọn { }
  • Dấu chấm phẩy: kết thúc câu lệnh
  • Ghi chú: // dòng, /* ... */
public class TenLop {
    public static void main(String[] args) {
        // code ở đây
    }
}

Giải thích hàm main

  • public: truy cập từ ngoài lớp
  • static: gọi mà không cần tạo đối tượng
  • void: không trả về giá trị
  • String[] args: tham số dòng lệnh

Quy tắc đặt tên & package

  • Tên lớp PascalCase: CustomerService
  • Tên biến/method camelCase: totalAmount, calculateTax()
  • Package chữ thường: com.company.app
  • Tên file trùng tên lớp public

Toán tử phổ biến

  • Số học: +, -, *, /, %
  • Gán: =, +=, -=...
  • So sánh: ==, !=, >, <, >=, <=
  • Logic: &&, ||, !
  • Điều kiện: ?: (ternary)
int a = 7, b = 3;
int sum = a + b;         // 10
boolean ok = (a > b) && (b > 0);  // true
String s = (a % 2 == 0) ? "chẵn" : "lẻ";

Câu lệnh if / else

int x = 10;

if (x > 0) {
    System.out.println("Số dương");
} else if (x == 0) {
    System.out.println("Bằng 0");
} else {
    System.out.println("Số âm");
}

Khối lệnh dùng { }. Một câu lệnh đơn có thể bỏ ngoặc, nhưng nên giữ cho rõ ràng.

Ví dụ: kiểm tra chẵn/lẻ & tìm số lớn hơn

// Chẵn / lẻ
int n = 15;
if (n % 2 == 0) {
    System.out.println("Chẵn");
} else {
    System.out.println("Lẻ");
}
// Max của hai số
int a = 5, b = 9;
int max;
if (a > b) {
    max = a;
} else {
    max = b;
}
System.out.println("Max = " + max);

Lỗi thường gặp (cảnh báo)

  • So sánh String: dùng .equals(), không dùng ==
  • Chia số nguyên: 5 / 2 == 2, muốn 2.5 dùng 5.0 / 2
  • Thiếu dấu chấm phẩy hoặc dấu ngoặc nhọn
  • Tên file ≠ tên lớp public → lỗi biên dịch
// Sai
String s1 = "Hi", s2 = new String("Hi");
System.out.println(s1 == s2);     // có thể false

// Đúng
System.out.println(s1.equals(s2)); // true

Nhập dữ liệu từ bàn phím (tham khảo)

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Nhập tên: ");
        String name = sc.nextLine();
        System.out.println("Xin chào " + name);
        sc.close();
    }
}

Bài tập về nhà

  1. Cài JDK (17/21) và một IDE. Gửi ảnh chụp java -version.
  2. Tạo chương trình in ra họ tên và tuổi (2 biến) theo mẫu: 
    Họ tên: ...
Tuổi: ...
  3. Viết chương trình nhập 2 số nguyên, in:
    • Tổng, hiệu, tích, thương (dạng số thực)
    • Số lớn hơn
    • Cả hai đều chẵn? (true/false)
  4. Nâng cao (tuỳ chọn): nhập điểm (0–10), in xếp loại:
    • ≥ 8.5: Giỏi; 7.0–8.4: Khá; 5.5–6.9: Trung bình; 0–5.4: Yếu

Đặt tên file và lớp đúng quy tắc. Format code gọn gàng.
